# ML-виз

Интерактивные визуализации алгоритмов машинного обучения. Вдохновлено
[Visualizing DBSCAN Clustering](https://www.naftaliharris.com/blog/visualizing-dbscan-clustering/)
Нафтали Харриса.

Чистая статика: vanilla JS + canvas, без сборки и без зависимостей. Работает офлайн.

## Запуск

```bash
./serve.sh          # поднимет http://localhost:8000
# или
python3 -m http.server 8000
```

Открыть `http://localhost:8000`. Можно и просто открыть `index.html` двойным
кликом — но через сервер корректнее (относительные пути, никаких сюрпризов с `file://`).

## Что готово

Все три используют общий набор датасетов (смайлик, кольца, сгустки, полумесяцы,
шум+сгусток) и единые управление и горячие клавиши:
`Space` — play/pause, `→` — шаг, `R` — сброс, `N` — новые данные, клик по полю — добавить точку.

- **DBSCAN** (`viz/dbscan/`) — плотностная кластеризация по шагам.
  Параметры: число точек, радиус ε, minPts. Показывает ядро / границу / шум и ε-окрестность активной точки.
- **K-Means** (`viz/kmeans/`) — кластеризация по центроидам.
  Чередование фаз assign → update, плавное скольжение центров с траекторией, монотонно падающая inertia.
  Три инициализации: **k-means++** (вероятностная ∝ D²), **farthest-first** (детерминированно самые далёкие, алгоритм Гонзалеса), случайная.
- **Иерархическая** (`viz/hierarchical/`) — агломеративная кластеризация.
  Linkage **ward** / average / single / complete (Ланс–Уильямс), дерево слияний на поле + **дендрограмма** (высота ветви = порог слияния, цвета синхронны с полем), срез на K.
- **Дерево решений** (`viz/dtree/`) — CART по шагам.
  Размеченные датасеты, осевые регионы границы решений, критерий Джини/энтропия, диаграмма дерева, train-accuracy (видно переобучение с ростом глубины).
- **Random Forest** (`viz/forest/`) — ансамбль деревьев.
  Bootstrap + случайные признаки (max_features), гладкая граница ансамбля (heatmap голосов) против «зубчатого» одиночного дерева; снижение дисперсии.

Общие модули supervised: `assets/js/datasets-labeled.js` (размеченные датасеты) и
`assets/js/tree-core.js` (CART: Джини/энтропия, bestSplit, buildTree, predict).

В планах (карточки «Скоро» на главной): градиентный спуск/бустинг, k-NN, регрессия / SVM.

## Структура

```
index.html              лендинг-галерея
assets/css/main.css     дизайн-система и галерея
assets/css/viz.css      общий лейаут страниц визуализаций
assets/js/datasets.js   общие генераторы датасетов (нормализованы в [0,1])
viz/dbscan/             index.html · dbscan.js (DBSCANStepper) · app.js
viz/kmeans/             index.html · kmeans.js (KMeansStepper) · app.js
viz/hierarchical/       index.html · hierarchical.js (HierarchicalStepper) · app.js
```

Каждая визуализация — одна папка: алгоритм вынесен в файл-степпер с методом
`step()`, отрисовка на canvas и UI — в `app.js`. Датасеты общие.

## Как добавить новую визуализацию

1. Создать папку `viz/<алгоритм>/` с `index.html`, подключив
   `../../assets/css/main.css` и `../../assets/css/viz.css`.
2. Логику алгоритма вынести в отдельный файл-степпер с методом `step()`,
   отрисовку и UI — в `app.js` (по образцу DBSCAN).
3. На главной (`index.html`) поменять карточку алгоритма с `card soon` на
   `card ready` и проставить ссылку.
